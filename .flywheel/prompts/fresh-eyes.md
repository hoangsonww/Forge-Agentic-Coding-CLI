# Fresh eyes on the bead graph

**Use with:** a **brand-new** Claude Code or Codex session. Fresh
sessions don't carry accumulated assumptions.

**When:** bead polishing has flatlined — the current session keeps
making the same small changes.

**Step 1.** Send this first:

---

First read ALL of the `AGENTS.md`, `CLAUDE.md`, `FLYWHEEL.md`, and
`README.md` files super carefully and understand ALL of them. Then
use your code-investigation mode to fully understand the Forge code,
its technical architecture, and its purpose. Use **ultrathink**.

---

**Step 2.** After it finishes, send:

---

We recently transformed a markdown plan
(`.flywheel/plans/<PLAN_FILE>.md`) into beads in
`.beads/beads.jsonl`. Review and analyze them. Check over each bead
super carefully — are you sure it makes sense? Is it optimal? Could
we change anything to make the system work better for users? If so,
revise the beads. It's a lot easier and faster to operate in "plan
space" before we start implementing these things! Use **ultrathink**.
