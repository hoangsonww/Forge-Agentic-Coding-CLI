# Operators

Eight canonical cognitive moves that show up across real Flywheel
sessions. Each operator is a drop-in prompt module you can paste into
any agent session.

Operators matter more than single prompts because they codify:

- **When** to apply the move.
- **What failure looks like.**
- **What output is expected.**

They're numbered in roughly the order they appear in a project's
lifecycle, but you can invoke any of them at any time.

| # | Operator                         | Use when                                               |
| - | -------------------------------- | ------------------------------------------------------ |
| 1 | `plan-first-expansion.md`        | Jumping to code before the plan is coherent            |
| 2 | `competing-plan-triangulation.md`| One model's biases are dangerous                       |
| 3 | `overshoot-mismatch-hunt.md`     | Review output looks shallow                            |
| 4 | `plan-to-beads-transfer-audit.md`| Beads are about to (or have just) left the plan behind |
| 5 | `convergence-polish-loop.md`     | Artifact has visible rough edges                       |
| 6 | `fresh-eyes-reset.md`            | Current session has done too many review rounds        |
| 7 | `fungible-swarm-launch.md`       | Beads are polished; time to launch                     |
| 8 | `feedback-to-infrastructure.md`  | Lessons have accumulated; turn them into artifacts     |
